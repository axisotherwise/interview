# 노드
<br>

### 노드
V8 엔진으로 빌드 된 자바스크립트 실행기입니다. <br>
싱글 스레드 논 블로킹 I/O 모델이며 이벤트 기반으로 작동합니다. <br>

<br>

### 노드 사용 이유
I/O 작업이 많은 경우 적합합니다. <br>
I/O 작업이란 파일 시스템 접근이나 네트워크 요청 작업등을 의미합니다. <br>

<br>

### 노드 작동 원리
V8 엔진과 libuv로 이루어져 있습니다. <br>

1. V8 엔진은 동기적인 자바스크립트 코드를 실행하며 CPU Bound 작업을 수행합니다.<br>

2. libuv는 비동기적인 I/O 작업을 수행합니다. <br>
libuv는 OS 커널과 스레드 풀로 이루어져 있습니다. <br>

* libuv는 비동기 작업이 들어올 경우 커널이 할 수 있는지를 판단합니다. <br>
* 커널이 할 수 없는 작업은 스레드 풀에게 위임합니다. <br>

<br>

### 노드 장단점
1. 장점 <br>
파일 시스템 접근 데이터베이스 접근 네트워크 요청 등 I/O 작업에 적합합니다. <br>
싱글 스레드이므로 컴퓨터의 자원을 적게 사용합니다. <br>
멀티 스레드에 비해 비교적 개발 난이도가 쉽습니다. <br>

* 실시간 스트리밍이나 채팅 주식 차트 서비스에 적합합니다. <br>

2. 단점 <br>
CPU 집약적인 작업에 적합하지 않습니다. <br>
싱글 스레드이므로 스레드가 죽으면 서버가 죽습니다. <br>

* 이미지나 비디오 처리 대규모 처리 서비스에 적합하지 않습니다. <br>

<br>

### 싱글 스레드 병렬
libuv 내부 이벤트 루프를 통해 비동기 방식으로 동시성 처리가 가능합니다. <br>

<br>

### 이벤트 기반 프로그래밍
특정 이벤트 발생에 의해 프로그램 흐름이 결정되는 패턴입니다. <br>
한 번에 하나의 코드를 순차적으로 실행하는 순차적 프로그래밍의 단점을 보완하기 위해 등장하였습니다. <br>

<br>

### 논 블로킹 I/O
이전 작업의 완료 여부와 관계없이 다음 코드가 실행됨을 의미합니다. <br>

<br>

### NPM YARN
자바스크립트 패키지 매니저를 의미합니다. <br>
패키지들의 설치와 의존성 관계 등을 안전하게 수행할 수 있게 해줍니다. <br>

* 성능 차이는 거의 없습니다.
* NPM은 하나의 패키지에 포함된 다른 패키지를 모두 설치하여 보안 이슈가 발생할 수 있습니다. 
* yarn은 yarn.lock 또는 package.json 파일에 있는 패키지들만 설치합니다.

<br>

### package.json
프로젝트의 정보와 설치된 모듈들의 의존성을 관리하는 파일입니다. <br>

<br>

### dependency devDependency
프로덕션에서 사용할 패키지와 개발 시 사용할 패키지를 의미합니다. <br>

* 런타임 단계에서 필요하면 dependency
* 컴파일 단계에서 필요하면 devDependency

<br>

### 익스프레스
노드의 내장 모듈인 http의 단점을 보완한 웹 프레임워크입니다. <br>

미들웨어, 라우팅 등을 사용해 짦은 시간안에 효율적인 개발이 가능합니다. <br>

<br>

### 미들웨어
요청과 응답 중간에 위치한 함수입니다. <br>
요청을 처리하기 위해 각자의 책임을 가지는 미들웨어를 조합하여 개발합니다. <br>
요청과 응답을 조작하여 기능을 추가하거나 요청을 필터링할 수 있습니다. <br>

<br>

### 라이브러리 프레임워크
1. 라이브러리 <br>
특정 기능을 수행하는 코드의 집합을 의미합니다. <br>

2. 프레임워크 <br>
일정한 형태와 필요한 기능을 갖춘 뼈대를 의미합니다. <br>

<br>

둘의차이는 제어 흐름이 누구한테 있는가입니다. <br>

* 라이브러리는 사용자가 코드의 흐름을 제어할 수 있습니다. <br>
* 프레임워크는 프레임워크에 의해 코드의 흐름이 제어됩니다. <br>

<br>

## SSR CSR 
1. SSR <br>
서버에서 페이지를 구성하여 사용자에게 보여줍니다. <br>
서버에서 모든 데이터를 매핑하므로 CSR보다 초기 페이지 구성 속도는 늦지만 전체적으로 사용자에게 보여주는 콘텐츠의 완료 시점은 빠르며 SEO 로직에 가산점이 있습니다. <br>

2. CSR <br>
클라이언트에서 페이지를 구성하여 사용자에게 보여줍니다. <br>
초기 페이지 구성 속도는 SSR 보다 빠르지만 서비스에 필요한 데이터를 추가로 서버에 요청해야하므로 전체적인 콘텐츠의 완료 시점은 SSR 보다 느립니다. <br>

<br>

### MVC 패턴
모델 뷰 컨트롤러로 이루어진 디자인 패턴입니다. <br>
애플리케이션의 구성 요소를 3가지 역할로 나누어 개발 프로세스에서 각각의 구성 요소에만 집중하여 개발할 수 있다는 점과 재사용과 확장성이 용이하다는 장점이 있습니다. <br>

* 서비스 규모가 커질수록 모델과 뷰의 복잡성이 증가한다는 단점이 있습니다. <br>

<br>

1. Model <br>
어떠한 동작을 수행할지 정의합니다. <br>
사용자가 편집을 원하는 모든 데이터를 가지고있어야합니다. <br>

<br>

2. View <br>
모델에게 질의해 결과값을 가져와 사용자에게 보여줍니다. <br>
모델이 가지고 있는 데이터를 저장해서는 안됩니다. <br>

<br>

3. Controller <br>
사용자 요청에 맞는 데이터를 모델에 요청해 결과를 View에 반영합니다. <br>

<br>

### body query params
1. body <br>
JSON XML Multi Form 데이터를 담습니다. <br>

2. query <br>
주소 ? 이후에 데이터를 담습니다. 엔드 구분자를 이용해 여러 개의 데이터를 담을 수 있습니다. <br>

3. params <br>
주소에 포함된 데이터를 담습니다. <br>

리소스를 고유하게 식별해야 한다면 params <br>
정렬이나 필터링을 해야한다면 query <br>
데이터의 양이 많고 이진 데이터일 경우 body <br>

<br>

### 객체 지향
객체들의 집합으로 프로그램의 상호 작용을 표현하여 데이터를 객체로 취급하여 객체 내부의 메서드를 활용하는 소프트웨어 방법론입니다. <br>

* 장점으로 상속 기능으로 코드를 재사용해 생산성이 증가합니다. <br>
* 단점으로 설계에 많은 시간이 소요되고 처리 속도가 다른 프로그래밍 방식에 비해 느립니다. <br>

<br>

1. 추상화 <br>
복잡한 시스템으로부터 핵심적인 개념 또는 기능을 추출해 내는 걸 의미합니다. <br>

<br>

2. 캡슐화 <br>
객체의 속성과 메서드를 하나로 매핑해 일부를 외부에 감추어 은닉하는 걸 의미합니다. <br>

* 은닉이란 다른 객체에게 자신의 정보를 숨기고 자신의 연산만을 통해 접근하는 걸 의미합니다. <br>

<br>

3. 상속성 <br> 
상위 클래스의 특성을 하위 클래스가 이어받아 재사용하거나 추가 확장하는 걸 의미합니다. 코드의 재사용과 유지 보수 측면에서 중요합니다. <br>

<br>

4. 다형성 <br>
하나의 클래스나 메서드가 상황에 따라 다양한 방법으로 동작하는 걸 의미합니다. <br>
